# WorldSettingsPlugin Project Description

## Overview
WorldSettingsPlugin is a Spigot/Paper Minecraft plugin that lets players configure world gameplay settings through an in-game chest GUI. Instead of using only commands or config files, authorized players use /worldsettings to open a menu and click options to toggle features, adjust numeric values, and cycle through preset modes.

The plugin currently keeps all settings in memory while the server is running. That means values update instantly in the GUI, but reset on server restart unless persistence is added later.

## Tech Stack
- Language: Java
- Build tool: Maven
- Server API: Spigot API (1.20.4 snapshot)
- Java target: 17 (set in Maven configuration)

## Top-Level Project Structure
- pom.xml - Maven build configuration and dependencies
- README.md - setup/build/test usage guide
- gitignore - ignored files for version control
- src/main/resources/plugin.yml - Bukkit/Spigot plugin metadata
- src/main/java/com/example/worldsettings/WorldSettingsPlugin.java - plugin entry point
- src/main/java/com/example/worldsettings/gui/SettingsGUI.java - GUI construction and layout logic
- src/main/java/com/example/worldsettings/listeners/GUIClickListener.java - click handling and setting updates
- src/main/java/com/example/worldsettings/settings/WorldSettings.java - state model for configurable settings
- target/ - compiled output and packaged artifacts (generated by Maven)

## File-by-File Description

### 1) pom.xml
Purpose:
- Defines the Maven project coordinates:
	- groupId: com.example.worldsettings
	- artifactId: world-settings-plugin
	- version: 1.0.0
- Sets compiler language level:
	- maven.compiler.source: 17
	- maven.compiler.target: 17
- Declares Spigot snapshot repository.
- Adds dependency on org.spigotmc:spigot-api:1.20.4-R0.1-SNAPSHOT with provided scope.

How it interacts:
- Used by Maven during mvn package to compile source files and build the plugin JAR.
- The provided dependency means the Minecraft server supplies Spigot classes at runtime.

### 2) src/main/resources/plugin.yml
Purpose:
- Registers plugin metadata for Spigot/Paper:
	- name, version, main, api-version, description, author.
- Declares command:
	- /worldsettings
- Declares permission:
	- worldsettings.use (default op).

How it interacts:
- Spigot reads this file at startup.
- main tells Spigot to instantiate WorldSettingsPlugin.
- Command and permission here map to the command handling in WorldSettingsPlugin.

### 3) src/main/java/com/example/worldsettings/WorldSettingsPlugin.java
Purpose:
- Main plugin class (extends JavaPlugin).
- Manages plugin lifecycle:
	- onEnable() initializes shared state and registers listeners.
	- onDisable() logs shutdown message.
- Handles /worldsettings command in onCommand(...).
- Stores singleton-like plugin instance and one WorldSettings object.

How it interacts:
- Creates WorldSettings instance used across GUI and listeners.
- Registers GUIClickListener so inventory clicks are captured.
- Calls SettingsGUI.openMainMenu(player) when command is executed.
- Exposes getWorldSettings() for other classes to read/update settings.

### 4) src/main/java/com/example/worldsettings/gui/SettingsGUI.java
Purpose:
- Builds the in-game inventory menu titled "Customize World Settings".
- Defines menu constants:
	- inventory size (54 slots)
	- exact slot numbers for each interactive setting item.
- Renders background, title item, divider, and all setting items.
- Uses helper methods to create items and ON/OFF visuals.

How it interacts:
- Pulls current values from WorldSettingsPlugin.getInstance().getWorldSettings().
- Reflects real-time setting values in item names and lore.
- Called when opening the menu and after each click update to redraw UI.

### 5) src/main/java/com/example/worldsettings/listeners/GUIClickListener.java
Purpose:
- Listens for InventoryClickEvent.
- Filters clicks so only this plugin menu is handled.
- Prevents players from moving GUI items (event.setCancelled(true)).
- Maps clicked slots to corresponding setting operations.
- Supports left-click and right-click behaviors for increase and decrease actions.
- Sends feedback messages and click sound.

How it interacts:
- Reads and mutates WorldSettings via plugin instance.
- Uses SettingsGUI slot constants to keep behavior aligned with GUI layout.
- Calls SettingsGUI.openMainMenu(player) after every valid click to refresh displayed values.

### 6) src/main/java/com/example/worldsettings/settings/WorldSettings.java
Purpose:
- Holds all configurable values in memory.
- Encapsulates business logic for updates:
	- toggles (boolean values)
	- sliders and ranged values with min and max bounds
	- cyclic values (difficulty and first blood moon day)
- Defines DifficultyLevel enum (EASY, MEDIUM, HARD) with cycle and display helpers.

How it interacts:
- Created once in WorldSettingsPlugin.
- Read by SettingsGUI when drawing item lore and status.
- Modified by GUIClickListener in response to player clicks.

### 7) README.md
Purpose:
- Human documentation for setup, build steps, server setup, usage, and troubleshooting.

How it interacts:
- Not used directly by runtime code.
- Helps developers and users run and test the plugin correctly.

### 8) target/ directory
Purpose:
- Maven-generated build output.
- Contains compiled classes and packaged artifact.

How it interacts:
- Generated from source, not hand-edited.
- Plugin JAR in this folder is copied to server plugins/ for runtime use.

## Runtime Interaction Flow

1. Server startup
- Spigot loads plugin.yml.
- Spigot creates WorldSettingsPlugin and calls onEnable().
- Plugin initializes WorldSettings and registers GUIClickListener.

2. Player command
- Player runs /worldsettings.
- WorldSettingsPlugin.onCommand(...) validates sender is a player.
- Plugin opens the menu via SettingsGUI.openMainMenu(player).

3. GUI render
- SettingsGUI reads current values from WorldSettings.
- GUI items are rendered to show current ON/OFF states and numeric values.

4. Player click
- GUIClickListener.onInventoryClick(...) receives click event.
- It confirms this is the plugin menu, cancels item movement, and maps slot to action.
- Listener updates WorldSettings using toggle, increase, decrease, and cycle methods.

5. Feedback and refresh
- Listener sends chat feedback and sound.
- Listener reopens menu through SettingsGUI so updated values are immediately visible.

## Design Notes
- Current state is in-memory only, so settings reset after restart.
- Two settings fields exist but are not currently represented in the GUI:
	- enhancedMobs
	- enhancedLootDrops
- Separation of concerns is clear:
	- WorldSettingsPlugin: lifecycle and command wiring
	- SettingsGUI: presentation
	- GUIClickListener: interaction control
	- WorldSettings: data and rules

## Summary
This project is a clean GUI-driven plugin architecture where command input opens a view, clicks trigger controller logic, and a shared settings model acts as the single source of truth. The classes are loosely coupled through clear responsibilities, making the plugin easy to extend with additional settings or persistence in future versions.
